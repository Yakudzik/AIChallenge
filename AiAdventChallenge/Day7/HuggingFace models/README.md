В приложение из Hugging Face добавлены 3 модели начала, середины и конца списка: MiniMax-M2.1, Llama-3-COT-ITA:featherless-ai, L3-8B-Stheno-v3.3-32K; по замерам ответов самая быстрая MiniMax-M2.1 (5696 ms) против 18258 ms и 23172 ms, по токенам: 350 / 471 / 141 соответственно; по качеству MiniMax-M2.1 дала корректный и связный ответ, Llama-3-COT-ITA:featherless-ai заметно медленнее и содержит неточности (дата/время), L3-8B-Stheno-v3.3-32K наиболее экономична по токенам, но короче и в большинстве случаев отвечает на английском - итог: быстрее всего MiniMax-M2.1, меньше всего токенов у L3-8B-Stheno-v3.3-32K, по качеству лидер MiniMax-M2.1.

## Возможности приложения
- Разбиение исходной задачи на подзадачи и маршрутизация по «экспертам».
- Генерация/сбор промежуточных результатов и финальная агрегация ответа.
- Изменение температуры в запросах (например, более «строго» или более «креативно»).
- Логи/отладочный вывод для понимания, какой «эксперт» что сделал (если включено в коде).
- Поддержка конфигурации через переменные окружения и/или файл с токенами.

## Стек технологий
- Python 3.10+ (рекомендуется 3.11+)
- Зависимости из `requirements.txt` (если файл присутствует в репозитории)
- (Опционально) `.env` / `python-dotenv` — для хранения и загрузки секретов

## Хранение токенов в отдельном файле
Токены хранятся в отдельном файле `tokens.txt` (формат `KEY=VALUE`, по одному на строку). Не коммитьте реальные ключи в репозиторий.

Какие токены могут понадобиться:
- `DEEPSEEK_API_KEY` — обязателен (без него приложение не стартует).
- `YANDEX_CLOUD_API_KEY`, `YANDEX_PROJECT_ID` (или `YANDEX_PROJECT`) + один из `YANDEX_PROMPT_ID` / `YANDEX_MODEL_ID` — опционально, если хотите провайдера Yandex.
- `CLAUDE_API_KEY` (или `ANTHROPIC_API_KEY`) — опционально, если хотите провайдера Claude.
- `HF_TOKEN` — опционально, если хотите провайдеры Hugging Face.
- `HF_MODEL_ID`, `HF_MODEL_TLAMA_ID`, `HF_MODEL_MAGNUM_ID` — опционально, чтобы переопределить модели Hugging Face.

Пример `tokens.txt`:
```txt
DEEPSEEK_API_KEY=...
# Yandex (опционально)
YANDEX_CLOUD_API_KEY=...
YANDEX_PROJECT_ID=...
YANDEX_MODEL_ID=...
# Claude (опционально)
CLAUDE_API_KEY=...
# Hugging Face (опционально)
HF_TOKEN=...
HF_MODEL_ID=...
HF_MODEL_TLAMA_ID=...
HF_MODEL_MAGNUM_ID=...
```

## Как развернуть и запустить на Windows
### 1) Подготовка окружения
Установите Python и проверьте версию:
```powershell
python --version
```

Создайте и активируйте виртуальное окружение:
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

### 2) Установка зависимостей
Если в репозитории есть `requirements.txt`:
```powershell
pip install -r requirements.txt
```

### 3) Настройка токенов
Создайте `.env` и добавьте ключи (см. раздел выше).

### 4) Запуск
Варианты:
- Через батник (проще всего):
  ```powershell
  .\startBot.bat
  ```
- Вручную через `uvicorn`:
  ```powershell
  uvicorn web_app:app --reload
  ```

После запуска откройте `http://127.0.0.1:8000/`.
